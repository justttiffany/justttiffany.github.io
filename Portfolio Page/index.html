Scripps Development Team Member

Scripps Institute of Oceanography, UCSD

April 2017 - June 2017

The images above depict the user interface of the CNN program. Users are able to easily select what they want to do and select files to test out the CNN with the given options.

I worked in a team of 4 people under the supervision of a Scripps researcher to continue the development of a Convolutional Neural Network that can categorize and classify recorded fish noises according to species. My role in the team was a front-end software developer. My task, along with one other team member, was to implement a user-friendly interface using Matlab's GUIDE tool for the CNN to improve usability. In addition, I developed and maintained a team website with one other team member to document progress: underwateracoustics.netlify.com

Task: Implement a user-friendly interface for the Convolutional Neural Network to improve usability.

Technologies/Languages/Development Tools Used: Matlab, Github

Process: We were given an existing code base to work with. My role, along with one other team member, was to come up with a user-friendly interface for the program so users of any background can easily use and navigate with data.

Here was my process through working on this task (quarter-long project):

  • Began looking through the existing code legacy to understand what each file did
  • Replicated the program to get an idea of how the program worked with the data passed in by users
  • Researched how to create a GUI in MATLAB and began implementation of task
  • Ported over the existing code base to the GUI program and modified the files to work towards a working interface

Reflection: Overall, the project was a great way to get experience on working in a team and gaining technical skills. I had never programmed in MATLAB, so that did give some challenges. We were well-monitored by the Scripps researcher by having bi-weekly check-ins to give an update on our progress. At the end of the quarter, we turned in the final product and it was used to demo at a science conference the next month! The images shown above is the end working product.

CaSGC/UCSD Intern

California Space Grant Consortium, UCSD

August 2016 - September 2016

I was an intern in the CaSGC/UCSD Summer Internship Academy Program sponsored by NASA during Summer 2016. I worked in a team of 8 people on the Arduino Computer Development and Programming Project to development an Arduino micro-controller for the launching of a near-space balloon. We spent a total of 5 weeks developing the micro-controller. I worked on the temperature and pressure sensors with one other team member.

Task: Develop an Arduino program for the micro-controller using temperature and pressure sensors that collected and recorded data for the Near-Space Ballooning Science Package.

Concepts Applied: Object-Oriented Design

Technologies/Languages/Development Tools Used: Arduino Programming Language, Arduino IDE, Soldering

Process: We were given the hardware component (microcontroller) of the project and had to code up a program that collected data of its surrounding environment using the hardware.

Along with one other team member, here was the process working towards this task:

  • Researched information on temperature and pressure sensors and similar existing programs
  • Began implementation of program that collected temperature and pressure data from the surrounding environment using the sensors
  • Connected all the hardware components onto the microcontroller to begin testing of the program
  • Extensive testing of the program by doing test cases and changing surrounding environments
  • Merged our program into the main team code base and tested to ensure integration was seamless

Reflection: This was my first project as an intern and working on a team. I learned a new language and got a taste of working with hardware components, which is different to me because I mainly am comfortable with software. Some difficulties include the pace of the work because we did not receive the necessary equipment until halfway through the internship, so I had to catch up to speed in implementation because the deadline was approaching. In the end, it was a successful project because it was used in the launching of a near-space balloon and teams reported back saying that the data was collected successfully.

Coursework Project
